# Essential Docker Commands

So far we've run containers in the foreground, blocking our terminal. Often, you'll need containers running in the background so you can do other tasks.

---

## Running Containers in the Background

You can run a container in detached mode (background) using the `-d` flag:

```bash
docker run -p 5000:80 -d nginx
```

**The `-d` flag** means "detached mode" – the container runs in the background and you get your terminal back.

### Common Use Cases

- Web servers (Nginx, Apache)
- Databases (MySQL, PostgreSQL, Redis)
- Background workers
- Microservices

---

## Viewing Running Containers

To see all running containers, use:

```bash
docker ps
```

The output shows:

- **CONTAINER ID**: Unique identifier for the container
- **IMAGE**: The image it's running from
- **NAMES**: Container name (random by default)
- **STATUS**: Whether it's running
- **PORTS**: Port mappings

Notice the container has a random name. You can assign your own name using the `--name` flag:

```bash
docker run -p 5000:80 --name my-nginx -d nginx
```

---

## Checking Container Logs

To see what's happening inside a container, view its logs:

```bash
docker logs [container_id or container_name]
```

**Examples:**

```bash
docker logs e065233cf1b8      # Using container ID
docker logs my-nginx           # Using container name
docker logs -f my-nginx        # Follow logs in real-time (like tail -f)
```

---

## Stopping a Container

To stop a running container:

```bash
docker stop [container_id or container_name]
```

**Note**: Stopping a container doesn't delete it – it just pauses it. The container still exists on your system.

---

## Removing Containers

### Manually Remove a Container

To delete a stopped container:

```bash
docker rm [container_id or container_name]
```

### Auto-Remove on Stop

You can automatically remove a container when it stops using the `--rm` flag:

```bash
docker run -p 5000:80 --rm nginx
```

When you stop this container, it's automatically deleted. Great for temporary containers!
