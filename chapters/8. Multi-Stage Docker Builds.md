# Multi-Stage Docker Builds: Smaller and Smarter Images

You've learned how to create Dockerfiles and optimize image sizes with Alpine images. But what if I told you there's an even better way? What if you could cut your image size **in half** or more?

Welcome to **multi-stage builds** – one of Docker's most powerful features for production deployments.

---

## The Problem: Build Dependencies You Don't Need

Let's say you're building a Node.js application. You need npm to install dependencies, but once your app is built, npm is just sitting in your image taking up space.

Here's a typical Dockerfile:

```dockerfile
FROM node:20-slim

WORKDIR /app

COPY package*.json ./

RUN npm install

COPY . .

CMD ["node", "index.js"]
```

The problem? Your final image contains:

- Node.js runtime (necessary)
- npm (not needed to run the app, only to build it)
- All the build dependencies npm installed
- Developer tools
- Temporary files from the build process

This makes your image **300 MB** when it could be much smaller.

### The Solution: Multi-Stage Builds

With multi-stage builds, you use **multiple `FROM` statements** – each one represents a different "stage." You build your app in one stage (with all the tools you need), then copy only the final output to a minimal stage.

It's like saying: "I'll use a huge kitchen with all the equipment to prepare the meal, then serve it on a simple plate with no extra stuff."

---

## How Multi-Stage Builds Work

Here's the basic concept:

```dockerfile
# Stage 1: Build Stage (we can name it)
FROM node:20-slim AS builder

WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build  # Builds your application

# Stage 2: Runtime Stage
FROM node:20-alpine  # Smaller base image

WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package*.json ./

CMD ["node", "dist/index.js"]
```

Notice:

- **`FROM ... AS builder`** — The first stage is named "builder"
- **`COPY --from=builder`** — Copy files from the builder stage, not from your computer
- The final image only includes what's needed to **run**, not to **build**

### Size Comparison

| Approach                | Size   | Why                          |
| ----------------------- | ------ | ---------------------------- |
| Single stage (standard) | 300 MB | Contains npm and build tools |
| Single stage (alpine)   | 200 MB | Still has npm and temp files |
| Multi-stage             | 120 MB | Only runtime dependencies    |

You just cut your image size by **60%**!

## Understanding the Commands: What Actually Happens

Let's break down each command so you understand exactly what's going on.

### Stage 1: The Builder (Building Your App)

```dockerfile
FROM node:20-slim AS builder
```

- Start with a **full Node.js image** that has npm and all build tools
- `AS builder` — Give this stage a name so we can refer to it later

```dockerfile
WORKDIR /app
COPY package*.json ./
RUN npm install
```

- Create a folder called `/app` inside the container
- Copy your `package.json` files from your computer into `/app`
- Run `npm install` — downloads everything you need (including build tools, dev dependencies, everything)

```dockerfile
COPY . .
RUN npm run build
```

- Copy all your source code (TypeScript, JavaScript, etc.)
- Compile your code (if using TypeScript, this turns `.ts` files into `.js` files)

**After Stage 1 is done, you have:**

- Your compiled app ready to run
- All the npm packages installed
- All the build tools and compilers (which you actually don't need anymore!)
- **Total size: ~300 MB**

### Stage 2: The Runtime (The Final Package)

```dockerfile
FROM node:20-alpine
```

- Start completely fresh with a **tiny Alpine image** (~50 MB)
- This is your final image – it should be small and clean

```dockerfile
WORKDIR /app
COPY --from=builder /app/dist ./dist
```

Here's the magic: **`COPY --from=builder`**

- Instead of copying from your computer, we copy from the builder stage
- We grab only the **compiled JavaScript** (the `.js` files in `/app/dist`)
- We put them in the current stage's `/dist` folder
- **The TypeScript compiler and all build tools? Left behind in the builder stage!**

```dockerfile
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package*.json ./
```

- Copy only the installed packages (runtime dependencies, no dev stuff)
- Copy the package files so Node knows what to run

- **Total size: ~95 MB**

## Example 1: The Simple "Hello Builder" - Your First Multi-Stage Build

Let's start **super simple**. This example is so straightforward that you'll understand multi-stage builds in minutes.

### The Idea

We're building a tiny Node.js app. Here's the emotional journey:

- **Stage 1 (Builder)**: We have a big kitchen with all tools. We write our code and prepare everything.
- **Stage 2 (Runtime)**: We serve the final meal on a clean plate. Nobody needs to see the kitchen mess!

### Step 1: Create Your Project Folder

```bash
mkdir simple-multi-stage && cd simple-multi-stage
```

What's happening: You're creating a folder for this experiment. Just like organizing your desk before starting work.

### Step 2: Create app.js

```javascript
const http = require("http");

const server = http.createServer((req, res) => {
  res.writeHead(200, { "Content-Type": "text/plain" });
  res.end("Hello from Multi-Stage Docker!\n");
});

server.listen(3000, () => {
  console.log("Server running on port 3000");
});
```

### Step 3: Create the Dockerfile

```dockerfile
# Stage 1: Builder
FROM node:20-slim AS builder

WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .

# Stage 2: Runtime
FROM node:20-alpine

WORKDIR /app

COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/app.js ./app.js
COPY --from=builder /app/package.json ./package.json

EXPOSE 3000
CMD ["npm", "start"]
```

### Step 4: Build and Run

```bash
docker build -t simple-app:v1 .
docker images | grep simple-app  # Should show ~45MB
docker run -p 3000:3000 simple-app:v1
```

Visit `http://localhost:3000`

## Example 2: Real-World Project - Express API with TypeScript

TypeScript needs compilation. Multi-stage keeps the compiler out of the final image.

### Project Structure

```
express-api/
├── Dockerfile
├── package.json
├── tsconfig.json
├── src/
│   └── index.ts
└── .dockerignore
```

### Step 1: Create tsconfig.json

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}
```

### Step 2: Create src/index.ts

```typescript
import express, { Request, Response } from "express";
import cors from "cors";

const app = express();
const PORT = 3000;

app.use(cors());
app.use(express.json());

app.get("/", (req: Request, res: Response) => {
  res.json({ message: "Express API with TypeScript" });
});

app.get("/api/users", (req: Request, res: Response) => {
  res.json([
    { id: 1, name: "Alice" },
    { id: 2, name: "Bob" },
  ]);
});

app.listen(PORT, () => {
  console.log(`API running on http://localhost:${PORT}`);
});
```

### Step 3: Create Dockerfile

```dockerfile
# Stage 1: Build and compile TypeScript
FROM node:20-slim AS builder

WORKDIR /app
COPY package*.json tsconfig.json ./
RUN npm ci
COPY src ./src
RUN npm run build

# Stage 2: Runtime only
FROM node:20-alpine

WORKDIR /app

COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package*.json ./

EXPOSE 3000
HEALTHCHECK --interval=30s --timeout=3s CMD node -e "require('http').get('http://localhost:3000')" || exit 1
CMD ["npm", "start"]
```

### Understanding Stage 2 (Runtime)

The **second stage** is where the magic happens:

1. **`FROM node:20-alpine`** - Fresh, minimal base image (~50MB)
2. **`COPY --from=builder /app/dist ./dist`** - Copy compiled JavaScript from builder
3. **`COPY --from=builder /app/node_modules ./node_modules`** - Copy runtime dependencies only
4. **`COPY --from=builder /app/package*.json ./`** - Copy package files

Result: **68% size reduction** (300MB → 95MB)

### Step 4: Build and Test

```bash
docker build -t express-api:v1 .
docker images | grep express-api

# Run it
docker run -p 3000:3000 express-api:v1

# Test
curl http://localhost:3000
```
